| Version | Supported          |
| ------- | ------------------ |

## Reporting a Vulnerability

We take the security of SplitRent seriously. If you believe you've found a security vulnerability, please follow these guidelines:

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to **[your-email@example.com]** with the subject line "Security Vulnerability - SplitRent".

Include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce the issue
- Affected components (frontend, smart contracts, etc.)
- Potential impact
- Any suggested fixes if applicable

### What to Expect

- **Acknowledgment**: You will receive an acknowledgment within 48 hours
- **Updates**: We'll provide updates every 7 days on progress
- **Resolution Timeline**: We aim to resolve critical issues within 30 days
- **Disclosure**: We coordinate responsible disclosure with the reporter

### Security Best Practices for Users

When using SplitRent:

1. Always verify you're on the correct URL
2. Use strong, unique passwords for wallet accounts
3. Keep your wallet extensions up to date
4. Never share your private keys or seed phrases
5. Use testnet for development and testing
6. Review all transaction details before signing

### Smart Contract Security

Our Soroban smart contracts undergo:

- Internal code reviews
- Community feedback during development
- Public audit reports when available

Audit reports will be published in the `docs/security/` directory.

### Bug Bounty Program

Currently, we don't have a formal bug bounty program. However, we greatly appreciate responsible disclosure and will acknowledge contributors who help improve our security (with permission).

## Security Updates

Security updates will be announced via:

- GitHub Security Advisories
- Release notes for patched versions
- Project README updates if necessary

Stay informed by watching the repository or checking releases regularly.
